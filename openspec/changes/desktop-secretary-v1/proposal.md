## Why

知识型PC用户（如建筑设计师）日常面临**待办事项与对应工作文件割裂**的痛点：任务记在待办工具里，文件散落在文件夹深处，每次处理任务都要反复查找资料，操作路径冗长、效率低下。现有待办工具均缺乏与本地文件系统的深度绑定能力，而现有桌面宠物/挂件类工具又缺乏真正实用的任务管理功能。

## What Changes

- 新增桌面常驻宠物窗口（透明置顶，可拖拽），承载情绪反馈动画与面板入口
- 新增悬浮待办面板，支持双区（置顶紧急区 + 常规区）卡片展示
- 新增待办快速创建流程（输入框 → 四象限选择器）
- 新增待办与本地文件/文件夹的关联绑定（拖拽 + 按钮）
- 新增智能文件匹配：点击待办自动匹配并打开关联文件夹内的核心文件
- 新增DDL截止时间设定与逾期状态计算
- 新增基于使用频率、象限权重、操作时效的动态排序算法
- 新增待办生命周期管理（完成归档、删除确认、缺失文件处理）
- 新增宠物动画状态机（8种状态，优先级驱动）
- 本地JSON数据持久化，无网络请求

## Capabilities

### New Capabilities

- `pet-system`: 桌面宠物窗口渲染、拖拽定位、位置记忆，以及基于任务状态的动画状态机（8种状态、优先级冲突规则）
- `todo-panel`: 悬浮待办面板的呼出/收起/钉住交互，双区卡片列表展示（置顶区 + 常规区）
- `todo-crud`: 待办的创建（快速输入 + 四象限选择）、查看、编辑标题、完成归档、删除（含二次确认）
- `file-association`: 待办与本地文件/文件夹的关联绑定（拖拽 + 文件选择器），关联记录的增删与缺失检测
- `smart-file-match`: 点击待办时基于标题关键词对关联文件夹进行模糊匹配，自动打开匹配文件及文件夹
- `ddl-management`: 截止时间的设定（快捷选项 + 自定义选择器）、清除、逾期状态实时计算
- `todo-sorting`: 双区排序规则实现（置顶区DDL升序、常规区Score公式动态排序）
- `data-persistence`: 本地JSON数据模型定义、启动加载、变更增量写入，数据存储于`%AppData%/DesktopSecretary/`

## Impact

- **新建应用**：Windows 10/11桌面客户端（Electron 或等效框架）
- **本地存储**：`%AppData%/DesktopSecretary/todos.json`，无外部依赖
- **系统API**：调用Windows Shell API打开文件夹/文件，调用系统文件选择器
- **性能约束**：常态动画CPU < 1%，内存 < 80MB，面板展开 < 100ms，智能匹配 < 200ms（< 500文件）
- **无网络请求**：全程离线，无后端服务
